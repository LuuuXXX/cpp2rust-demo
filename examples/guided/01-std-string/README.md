# 🔧 引导支持示例 01：`std::string` 参数与返回值

**汇总统计类别：🔧 引导支持**（工具生成签名正确的 C++ shim 原型，用户必须填写函数体）

---

## 背景

hicc 没有 `std::string` 的 ABI 支持——它无法将 `std::string` 安全地跨越 C++/Rust 边界传递。
cpp2rust-demo 在遇到含 `std::string` 参数或返回值的方法时：

1. **跳过**该方法，在接口报告中记录 `hicc_limitation`。
2. 在 `meta/operator_shims.hpp` 中**自动生成 C-string shim 签名**（函数体留空，等待用户填写）。
3. 在 `entry.rs` 中生成对应的 Rust 绑定骨架（已可编译，指向 shim 函数名）。

用户只需在 `operator_shims.hpp` 中填写 C++ shim 函数体（处理缓冲区/生命周期），
然后在 `hicc::cpp!` 宏中引入，即可完成绑定。

---

## C++ 源码（`string_utils.hpp` / `entry.cpp`）

```cpp
// string_utils.hpp
#include <string>

class StringProcessor {
public:
    explicit StringProcessor(const char* prefix);

    // ── 以下方法含 std::string → 被跳过，生成 shim 建议 ──────────────
    std::string transform(const std::string& input) const;
    int         count_chars(const std::string& text, char c) const;
    std::string join(const std::string& a, const std::string& b) const;

    // ── 以下方法只有 POD / C-string → 自动提取 ✅ ─────────────────────
    const char* prefix() const;
};
```

## 用户填写 shim 函数体

参考 `operator_shims.hpp` 中的注释，逐一填写函数体，例如：

```cpp
inline int string_processor_transform(
    const StringProcessor& self,
    const char* input,
    char* out_buf,
    int   out_size)
{
    std::string result = self.transform(std::string(input));
    int n = std::min((int)result.size(), out_size - 1);
    std::memcpy(out_buf, result.c_str(), n);
    out_buf[n] = '\0';
    return n;
}
```

然后在项目的 `hicc::cpp!` 宏中引入该头文件（详见 `docs/hicc-usage.md`）。

## 转换流程手册

```
C++ 方法（含 std::string）
    │  clang AST 解析
    ▼
CXXMethodDecl（参数含 std::string）
    │  is_supported_cpp_type 检查
    ▼
std::string 不被 hicc 支持 → SkipCategory::HiccLimitation
    │  codegen
    ▼
meta/operator_shims.hpp  ─── C++ shim 签名骨架（函数体为 TODO）
entry.rs         ─── Rust 绑定骨架（含 shim ops）（已可编译）
meta/init-interface-report.md ─── 跳过记录
    │
    │  用户填写 shim 函数体
    ▼
operator_shims.hpp（完整实现）
    │  在 hicc::cpp! 中引入
    ▼
可编译的 Rust → C++ std::string 绑定
```

## 场景解析

| 步骤 | 工具行为 | 用户操作 |
|------|---------|---------|
| `init` | 检测 `std::string` 参数/返回，标记 HiccLimitation | 无 |
| codegen | 生成 shim 签名骨架（空函数体）+ Rust 骨架 | 无 |
| 填写 shim | — | 在 `operator_shims.hpp` 中实现缓冲区/生命周期管理 |
| 引入 shim | — | 在 `hicc::cpp!` 中 `#include "operator_shims.hpp"` |
| 使用 | 正常调用 `string_processor_transform(...)` | 实现业务逻辑 |

---

## 限制说明

| 限制 | 说明 |
|------|------|
| 缓冲区大小 | 输出 `char*` 缓冲区大小需调用方预分配；工具无法推断合适大小 |
| 生命周期 | C++ 返回的 `std::string` 临时对象在 shim 函数返回后销毁；必须在函数内完成复制 |
| `std::string` 字段 | 类成员变量为 `std::string` 时同样不支持直接访问，需额外 shim |
