# 完整特性支持全景表

> 本文档对应示例已按汇总统计类别组织于 [`examples/`](../examples/README.md) 目录下，每个特性均有实际用例（C++ 源码、转换后 Rust 代码、转换流程手册）。

## 图例
- ✅ **完全自动**：工具生成代码，直接可编译使用，零人工介入
- ⚙️ **半自动**：工具生成可用骨架，用户需解注释或加 CLI flag
- 🔧 **引导支持**：工具生成签名正确的 C++ shim 原型/接口骨架，用户**必须填写函数体**
- ⚠️ **条件支持**：满足特定前置条件后重跑工具可自动化
- ❌ **不支持**：hicc 硬限制，无论工具如何改进都无法绑定

## 一、自由函数类

| C++ 特性 | hicc | cpp2rust-demo | 输出位置 | 用户需做什么 |
|---------|:----:|:-------------:|---------|------------|
| 普通自由函数 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无 |
| `inline` 函数 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（`inline` 对工具透明，与普通函数相同，[示例](../examples/features/01-inline-functions/README.md)） |
| 函数重载 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（自动 `_2`/`_3` 后缀） |
| 命名空间函数 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无 |
| 默认参数（忽略尾部参数） | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（[示例](../examples/features/02-default-params/README.md)） |
| 忽略返回值 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无 |
| 捕获 C++ 异常 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（`hicc::Exception<T>` 自动包装） |
| `va_list` 参数（最后参数） | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（生成 `unsafe fn foo(args, ...)` 绑定，[示例](../examples/features/04-va-list/README.md)） |
| 模板函数（有显式特化） | ⚠️ 部分 | ✅ 完全自动 | `free/fn_*.rs` | 无 |
| 模板函数（无显式特化） | ⚠️ 部分 | ⚠️ 条件支持 | — | 在 entry.cpp 添加 `template int foo<int>(int);` 显式实例化后重跑 |
| `...` C 风格 variadic | ⚠️ 部分 | ❌ 不支持 | — | 手写固定参数 C++ 包装函数后再绑定（hicc 限制：参数/返回值不能含 C++ 类类型） |
| 函数指针参数 | ❌ | 🔧 引导支持 | 接口报告 | 根据报告中的虚函数接口骨架，手写 C++ 接口类 + Rust `impl XxxHandler for MyStruct` |
| `std::string` 参数/返回 | ❌ 直接 | 🔧 引导支持 | `meta/operator_shims.hpp` | 填写 shim 函数体（C++ 缓冲区/生命周期管理），然后在 `hicc::cpp!` 中引入 |
| `std::function`/lambda 参数 | ✅ hicc支持 | 🔧 引导支持 | 接口报告 | 根据报告骨架手写 C++ 接口类 + `@make_proxy` + Rust impl |
| `auto`/`decltype` 返回类型 | ❌ | ❌ 不支持 | — | 手写显式返回类型的 C++ 包装函数后再绑定 |
| 友元函数 | ❌ | ❌ 不支持 | — | 以普通自由函数形式重写 shim 后绑定 |

| 混合类（纯虚 + 普通方法） | ✅ | ✅ 完全自动 | `method/mtd_*.rs` | 无（普通方法正常提取，纯虚方法生成 companion interface） |
| `@make_proxy` 反向绑定 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（全纯虚类自动生成） |
| 方法模板（类内函数模板） | ❌ | ❌ 不支持 | — | 针对每个具体实例化手写独立 shim 函数 |
| 运算符重载 | ❌ | 🔧 引导支持 | `meta/operator_shims.hpp` + `free/shim_ops.rs` | 填写 `operator_shims.hpp` 中每个运算符的 C++ 函数体，然后在 `hicc::cpp!` 中引入；Rust 侧 `shim_ops.rs` 绑定骨架已就绪 |
| 类成员变量（实例字段） | ✅ | ✅ 完全自动 | `method/mtd_*.rs` | 无（`#[cpp(field)]` 读写访问器自动生成，[示例](../examples/features/07-instance-fields/README.md)） |
| 类静态成员变量 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（`#[cpp(data)]` 自动生成，[示例](../examples/features/06-static-members/README.md)） |
| 全局变量 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（[示例](../examples/features/05-global-vars/README.md)） |

## 三、继承与多态类

| C++ 特性 | hicc | cpp2rust-demo | 输出位置 | 用户需做什么 |
|---------|:----:|:-------------:|---------|------------|
| public 单继承 | ✅ | ✅ 完全自动 | `method/mtd_*.rs` | 无（`class Derived: Base` 语法，支持 upcast） |
| `dynamic_cast` | ✅ | ⚙️ 半自动 | `free/dynamic_casts.rs` | 解注释需要的 cast 对；所有 downcast 绑定骨架已自动生成到 `free/dynamic_casts.rs` |
| Rust 实现 C++ 抽象接口 | ✅ | ✅ 完全自动 | `free/fn_*.rs` | 无（骨架已生成，只需 `impl XxxTrait for MyStruct`） |
| 多重继承（运行时语义） | ❌ | ⚠️ 骨架生成（无法运行） | `method/mtd_*.rs` | 工具提取所有 public 基类并生成 `class C: A, B` 骨架；hicc 不支持多重继承运行时语义，骨架**无法编译**；需手写 C++ 委托包装后单继承绑定 |
| 虚继承（菱形继承） | ❌ | ⚠️ 检测并报告 | 接口报告（`⚠️ Virtual bases (skipped)` 警告） | 工具自动检测并跳过虚基类，接口报告列出警告；hicc 不支持虚继承，需重新设计 C++ 层接口规避 |

## 汇总统计

| 层级 | 数量 | 说明 | 对应示例 |
|------|:---:|------|---------|
| ✅ **完全自动**（零人工） | **~35项** | 工具直接生成可编译绑定 | [`examples/simple/`](../examples/simple/README.md)、[`examples/class/`](../examples/class/README.md)、[`examples/rapidjson/01-enum/`](../examples/rapidjson/01-enum/README.md)～[`06-inheritance/`](../examples/rapidjson/06-inheritance/README.md)、[`08-multi-tu/`](../examples/rapidjson/08-multi-tu/README.md) |
| ⚙️ **半自动**（解注释或加 flag） | **2项** | `@dynamic_cast`、placement new；加 flag 可完全自动化 | [`examples/semi-auto/01-dynamic-cast/`](../examples/semi-auto/01-dynamic-cast/README.md)、[`examples/semi-auto/02-placement-new/`](../examples/semi-auto/02-placement-new/README.md) |
| ⚠️ **条件支持**（重跑可自动化） | **4项** | 无别名模板类、无别名容器参数、无特化函数模板、`Pod<T>` 模板参数 | [`examples/conditional/01-template-no-alias/`](../examples/conditional/01-template-no-alias/README.md)、[`examples/conditional/02-function-template/`](../examples/conditional/02-function-template/README.md) |
| 🔧 **引导支持**（签名已生成，函数体必须手写） | **5项** | 运算符重载、`std::string`、`std::function`/lambda、函数指针、`va_list` C风格variadic | [`examples/rapidjson/07-operator-shim/`](../examples/rapidjson/07-operator-shim/README.md)、[`examples/guided/01-std-string/`](../examples/guided/01-std-string/README.md)、[`examples/guided/02-std-function/`](../examples/guided/02-std-function/README.md)、[`examples/guided/03-function-pointer/`](../examples/guided/03-function-pointer/README.md) |
| ❌ **不支持**（hicc 硬限制） | **6项** | 析构函数显式绑定、方法模板、`auto`/`decltype`、友元函数、私有析构不生成、多重继承运行时语义和虚继承（工具生成骨架/警告但 hicc 本身不支持） | — |

> **核心结论**：cpp2rust-demo 对 hicc 已支持特性的工具层覆盖已经相当完整。**真正的人工介入边界**不是工具能力不足，而是**业务语义无法推断**——运算符的比较/赋值逻辑、`std::string` 的缓冲区管理策略、`std::function` 回调的具体实现，这些内容工具永远无法替代人来填写。工具已经做到了"给出签名正确的壳，用户只需填充业务逻辑"这一层，这是合理的能力边界。
